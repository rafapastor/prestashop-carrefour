# Security policy

## Supported versions

Until v1.0.0 is released, only `main` is supported. After v1.0.0, we support the latest minor version and the previous minor version with security fixes.

## Reporting a vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

The preferred channel is GitHub's **Private Vulnerability Reporting** — on this repo, go to *Security → Report a vulnerability*. GitHub notifies the maintainer privately and keeps the thread confidential until a fix is ready.

If you can't use the GitHub flow, email **carrefour@smart-shop-ai.com** instead with:

- Description of the vulnerability.
- Steps to reproduce.
- Affected version(s).
- Impact assessment (what can an attacker do?).
- Optional: suggested fix.

You'll get an acknowledgement within **72 hours** and a plan within **7 days**. Fixes for critical issues are prioritised and released as patch versions.

If you report a valid issue, you'll be credited in the release notes (unless you prefer to stay anonymous).

## Out of scope

- Vulnerabilities in PrestaShop core itself — report to the [PrestaShop security team](https://www.prestashop-project.org/security/).
- Vulnerabilities in third-party dependencies — report upstream; we'll update once a patch is available.
- Social engineering or physical attacks.

## Disclosure policy

Coordinated disclosure: we agree on a timeline before publishing. Default embargo: **30 days** from first patched release, unless the issue is actively exploited in the wild.
